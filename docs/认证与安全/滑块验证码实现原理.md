## 验证流程

### 阶段 1: 验证码生成

```typescript
// 1. 生成随机位置
const correctX = Math.floor(Math.random() * 210) + 50; // 50-260px
const y = Math.floor(Math.random() * (155 - 40));       // 随机垂直位置

// 2. 创建背景图
const background = new Jimp(310, 155, 0xe8e8e8);

// 3. 创建滑块（带圆形缺口）
const slider = new Jimp(44, 44, 0x00000000);
slider.circle(22, 22, 16, 0x4a90ef);   // 圆形轮廓
slider.circle(22, 22, 14, 0x00000000); // 中心挖空

// 4. 从背景图裁剪对应区域
const blockImage = background.clone().crop(correctX, y, 40, 40);

// 5. 粘贴到滑块上
slider.composite(blockImage, 2, 2);

// 6. 保存图片到文件系统
await fs.writeFileSync(`${captchaId}_background.png`, background);
await fs.writeFileSync(`${captchaId}_slider.png`, slider);

// 7. 存储正确位置到Redis
await cache.set(`captcha:slider:${captchaId}`, { correctX }, 300);
```

### 阶段 2: 用户拖动

```
用户操作流程：
1. 前端显示背景图和滑块
2. 用户按住滑块向右拖动
3. 前端记录拖动过程：
   - x: 拖动距离 (像素)
   - t: 拖动时间 (毫秒)
   - track: 拖动轨迹数组
4. 用户松开滑块
```

### 阶段 3: 后端验证

```typescript
async verifySliderCaptcha(captchaId: string, captchaResult: string): Promise<boolean> {
  // 1. 从Redis获取正确位置
  const stored = await this.cache.get<SliderCaptcha>(captchaKey);
  const correctX = stored.correctX; // 100

  // 2. 解析用户拖动结果
  const { x, t, track } = JSON.parse(captchaResult);

  // 3. 距离验证（允许5px误差）
  const diff = Math.abs(x - correctX); // |102 - 100| = 2
  if (diff > 5) return false; // ✅ 通过

  // 4. 时间验证
  if (t < 300 || t > 60000) return false; // 防止瞬移或挂起

  // 5. 速度验证
  const speed = x / t; // 102 / 1500 = 0.068 px/ms
  if (speed < 0.5 || speed > 8) return false;

  // 6. 轨迹验证（斜率变化）
  const slopeChanges = this.calculateSlopeChanges(track);
  if (slopeChanges < 0.1) return false; // 太平滑可能是脚本

  // 7. 验证通过，清理资源
  await this.cache.del(captchaKey);
  await this.cleanupCaptchaImages(captchaId);

  return true;
}
```

## 防作弊机制

### 1. 距离误差范围

| 偏差 | 说明 |
|------|------|
| ≤ 5px | 视为正常人体操作误差 ✅ |
| > 5px | 判定为错误 ❌ |

### 2. 时间限制

| 时间 | 说明 |
|------|------|
| < 300ms | 瞬移，脚本行为 ❌ |
| 300-60000ms | 正常人类拖动 ✅ |
| > 60000ms | 拖动过久，可能是挂起 ❌ |

### 3. 速度范围

```
理想速度 = 距离 / 时间

正常范围：
- 最小速度: 0.5 px/ms (太慢)
- 最大速度: 8 px/ms (太快)

计算示例：
- 快速拖动: 200px / 300ms = 0.67 px/ms ✅
- 正常拖动: 150px / 2000ms = 0.075 px/ms ✅
- 瞬间拖动: 200px / 50ms = 4 px/ms ✅
- 脚本拖动: 200px / 1000ms = 0.2 px/ms ✅
```

### 4. 轨迹分析

```
人类拖动轨迹（有抖动）：
X轴位置: ────┬───────┬────────┬───────
          98  102     100    99     101

斜率变化:    +4   -2    -1    +2
              ↓    ↓     ↓     ↓
绝对值和: 4 + 2 + 1 + 2 = 9
平均斜率变化: 9 / 3 = 3.0 ✅ (大于0.1)

机器拖动轨迹（完美直线）：
X轴位置: ────100────────100────────100

斜率变化:    0       0       0
              ↓       ↓       ↓
绝对值和: 0 + 0 + 0 = 0
平均斜率变化: 0 / 2 = 0 ❌ (小于0.1)
```

## 性能优化

| 优化项 | 实现方式 |
|--------|----------|
| 图片生成 | 使用Jimp异步处理，不阻塞主线程 |
| 验证码存储 | Redis缓存，自动过期 |
| 图片文件 | 验证成功后立即删除，节省磁盘空间 |
| 并发处理 | 每个验证码独立UUID，支持高并发 |

## 安全性评估

| 攻击方式 | 防御措施 |
|----------|----------|
| 简单脚本 | 时间 + 速度验证 |
| 图像识别 | 随机装饰图案 |
| 重放攻击 | 单次使用 + 过期清理 |
| 暴力破解 | 距离误差限制 |
| 自动化 | 轨迹斜率变化分析 |

## 总结

滑块验证码的核心是：
1. **图片生成**: 随机位置 + 裁剪拼图
2. **后端验证**: 多维度验证（距离、时间、速度、轨迹）
3. **资源管理**: 验证后立即清理
4. **防作弊**: 模拟人类行为特征
